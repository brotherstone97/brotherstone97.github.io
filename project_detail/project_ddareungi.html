Model Server

Summary

3학년 2학기 ‘IT창업실습’과목의 기말과제입니다.
2021년 11월 DACON에서 열린 따릉이 대여량 예측 경진대회의 데이터와 기상청 단기예보 Open API를 활용하여 내일의 따릉이 대여량 예측모델을 만들고 그 결과값을 웹페이지에 표시하여 사용자가 참고할 수 있도록 한 프로젝트입니다.

Background

이번 수업시간에 배운 데이터 사이언스 개념을 그 전에 배워왔던 프론트엔드, 백엔드를 내용과 결합하여 기말과제를 진행하면 좋겠다는 생각을 했습니다. 그래서 배정된 팀원들과 합의해 각자 희망하는 역할을 맡아서 했고 저는 ML과 Django에 관심이 많았기에 모델링과 REST API서버를 구현을 맡아서 하게되었습니다.

Detail

DACON에서 제공한 데이터는 2018년~2020년의 기상청 단기예보(풍향, 하늘 상태, 강수확률 등)를 feature로, 따릉이 대여량을 label로 정의하고있습니다. 우선 EDA를 거쳐 쓸모없는 feature는 제거하고 일교차 등 기존 feature에서 새로운 feature를 만들어냈습니다. 그리고 미세먼지 농도와 자전거 대여량의 상관관계도 분명 있을 것으로 판단하여 train데이터의 datetime과 일치하는 일평균 미세먼지 통계자료를 open API를 통해 csv파일로 가져와 기존 train데이터와 merge하여 데이터를 보강했습니다. 위 데이터를 기반으로 sklearn의 xgboost라이브러리를 통해 모델을 생성하였고 당장 내일의 대여량을 예측하기 위해선 오늘날짜의 train데이터와 동일한 feature값이 필요하기 때문에 경진대회 측에서 제공한 날씨데이터와 같은 종류의 데이터를 공공데이터포털에서 찾아 Open API로 받아왔습니다. 그렇게 feature값으로 필요한 데이터만 필터링하여 Django 서버에 POST메서드로 보내면 서버 내 탑재된 모델을 거쳐 예측값을 리턴받는 원리로 프로젝트를 완성했습니다.
